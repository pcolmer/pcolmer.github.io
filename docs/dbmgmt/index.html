Database management

Possible database management tasks are:

  • Exporting a database as a GEDCOM file or a Relative History database.
  • Renaming a database.
  • Deleting a database.

If using Windows Phone/Mobile, you initiate any of these tasks by opening the desired database and then tapping on the appropriate button on the app bar.

For other devices, you select the desired database on the main screen and then tap on the appropriate button on the app bar.

Deleting a database

If you choose to delete a dabase from the app, any image files associated with the database that are stored in Application storage (see Media management for more details) are also deleted. Image files stored elsewhere are not affected.

Exporting as a database

Exporting as a Relative History database creates a file that can quickly be imported into another copy of Relative History. If there are any image files associated with the database that are stored in Application storage (see Media management for more details) then these are included in the created file. For all other image files, the database just remembers the filename and it is necessary to add the relevant sources after the database has been imported so that the app can find the images again.

Exporting as a GEDCOM file

Please note that in the current version, exporting a database as a GEDOM file exports everything; a future version will provide options such as excluding living people.
